Mức INFO: Dùng để ghi lại các giao dịch thành công (nạp tiền, rút tiền). Thông tin này quan trọng với nghiệp vụ ngân hàng để đối soát. Ví dụ: "Nạp tiền vãng lai thành công. Số tiền: 500.0".

Mức DEBUG: Dùng để theo dõi chi tiết luồng chạy của chương trình, giúp lập trình viên dễ dàng trace code trong quá trình phát triển (ví dụ: bắt đầu đọc file, thêm khách hàng mới).

Mức ERROR: Dùng để ghi lại các ngoại lệ (Exceptions) và lỗi hệ thống (như rút tiền quá số dư, lỗi nạp số tiền âm). Kèm theo đó là stack trace để dễ dàng khoanh vùng lỗi.
